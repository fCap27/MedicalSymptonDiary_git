Medical Symptom Diary

Medical Symptom Diary è una web application pensata per il settore medico-sanitario, che consente ai pazienti di registrare i propri sintomi nel tempo e di prenotare una visita medica allegando automaticamente il diario dei sintomi.
L’applicazione prevede anche una sezione amministrativa, che permette alla struttura sanitaria di gestire le prenotazioni e consultare i dati inseriti dagli utenti.
Il progetto è stato sviluppato come esercizio pratico di progettazione e realizzazione di un sistema web moderno basato su architettura client–server e API RESTful.
Questo progetto ha lo scopo di dimostrare:
  -la progettazione di un sistema web completo
  -l’uso di API RESTful
  -la separazione tra frontend e backend
  -l’utilizzo di Docker per rendere l’applicazione riproducibile
  -la gestione di ruoli utente differenti (paziente e amministratore)
Il codice è organizzato in modo modulare per facilitare la manutenzione e possibili estensioni future.
